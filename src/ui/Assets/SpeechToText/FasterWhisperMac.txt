Faster Whisper Mac runs SYSTRAN's "faster-whisper" Python package
(https://github.com/SYSTRAN/faster-whisper) via python3 on macOS (Apple Silicon and Intel).

Why use it: CTranslate2's Whisper decoding gives noticeably better results in some
languages (e.g. Arabic) than the other local engines. The trade-off is speed - CTranslate2
has no Apple GPU backend, so this engine always runs on the CPU and is slower than
MLX Whisper. To compensate, Subtitle Edit's helper uses faster-whisper's batched
inference automatically (typically 2-4x faster than sequential decoding, identical
model weights and accuracy) and one CTranslate2 thread per CPU core.

Installation:

        pip3 install faster-whisper

Subtitle Edit detects the installation by running: python3 -c "import faster_whisper"
(several Python installs are probed: Homebrew, python.org, pyenv, system).

Models:

        tiny, base, small, medium, large-v2, large-v3, large-v3-turbo

        Models are downloaded automatically from Hugging Face on first use and cached in
        ~/.cache/huggingface/hub - no manual model download is needed. A local model path
        or a Hugging Face repo id can also be typed in directly.

        For Arabic and other accuracy-sensitive work, large-v3 is the best pick;
        large-v3-turbo trades a little accuracy for roughly double the speed.

How it runs:

        python3 faster_whisper_transcribe.py --audio <audio file> --model <model>
                --output-format srt --output-dir <audio folder> [--language <code>]
                [--task translate] [extra parameters]

        The SRT result is written next to the input audio file and loaded automatically.

Subtitle segmentation: raw Whisper segments make poor subtitles (lines too long or
too short, boundaries ignoring sentences). Like Faster-Whisper-XXL's --standard mode
on Windows, word-level timestamps are requested and cues are rebuilt: breaking at
sentence-ending punctuation and silence gaps, balance-splitting oversized sentences
at commas, with each cue timed from its first to its last word.

Extra parameters (add in the advanced-settings parameters box):

        --max-cue-chars N
                Max characters per cue before a forced split (default: 84, two 42-char lines).
        --max-cue-duration SECONDS
                Max seconds per cue before a forced split (default: 6).
        --raw-segments
                Write Whisper's raw segments instead of the rebuilt cues.

        --compute-type {auto,int8,int16,float32}
                CTranslate2 quantization (default: auto). int8 is roughly 2x faster on CPU
                with near-identical accuracy - the best speed knob if large-v3 feels slow.
        --batch-size N
                Batched-inference batch size (default: 8). 0 or 1 disables batching.
                Higher values use more RAM; 8-16 is a good range on Apple Silicon.
        --cpu-threads N
                CTranslate2 CPU threads (default: 0 = one per core).
        --device {auto,cpu}
                CTranslate2 device (default: auto). There is no Apple GPU (Metal) backend
                in CTranslate2, so on a Mac this always resolves to the CPU.
